# 设计模式

## 1. 类之间的关系

1.1 **单项关联**：实箭头；A类属性是B类

1.2 **双向关联**：直线；A类，B类属性互相包含

1.3 **自关联**：实箭头；自己的属性包括自己

1.4 **聚合关系**：空菱形；对象相对于整体是独立的（计算机和硬件）

1.5 **组合关系**：实菱形；整体和对象不可分（头和嘴巴）

1.6 **依赖关系**：虚线实箭头；A类方法使用了B类方法；

1.7 **继承( 泛化 )关系**：空箭头；父类与子类的关系

1.8 **实现关系**：虚线空箭头；接口与实现类的关系

## 2. 面对对象的设计原则

2.1 **单一职责原则**：类负责的职责必须单一，有利于管理

2.2 **开闭原则**：对扩展开放，对修改关闭。多使用抽象类和父类，
	这样增添新功能时不用改变代码，只需增添子类/实现类

2.3 **里氏替换原则**：父类能使用的地方，就可以用其子类替换。
	（长方形和正方形的组成依赖关系时违法此原则，因为正方形
		和长方形的法则不同，应创建一个四边形的父类）

2.5 **依赖倒转原则**：要对抽象类和接口编程，而不是对具体类编程

2.6 **接口隔离原则**：接口过大时要分割成更小的接口；避免具体类实现	接口时多写不需要的方法

2.7 **合成复用法则**：先考虑使用组合和聚会关系，其次才考虑继承关系
	继承的缺点：

	- 破环类的封装性，父类的实现细节暴露给子类
	- 子类和父类的耦合性高，父类的改变会影响子类
	- 限制了复用的灵活性

2.8 **迪米特法则**：如果类无直接通信，不应直接互相调用，让第三方
	（经纪人作为粉丝，商务公司与明星的中间人）

​	朋友的定义：

```
- 对象本身
- 对象的成员对象
- 对象创建的对象
- 对象的方法要用的对象
- 存在关联，聚合，组合关系的对象
```

## 3. 创建型模式

### 3.0 简单工厂模式

**结构**： 

- 抽象产品：抽象类/接口
- 具体产品：子类/实现类
- 具体工厂：接受标识符(如字符串，数字)，返回的是抽象产品的类，但对象是具体产品，使名称统一

**缺点**：每多一个具体产品，具体工厂要添加一个 if else/swith 语句来
	返回对应的产品，违反了开闭原则

### 3.1 工厂方法模式

**结构**：在简单工厂模式上，加了一个抽象工厂，提供方法模板的接口，
	每个具体工厂只负责一具体产品；

**优缺点**：相比简单工厂模式满足了开闭原则，但增加了系统复杂度
	每加一个产品都要多一个具体产品和一个具体工厂

### 3.2 抽象工厂模式

**结构**：在工厂方法模式基础上，抽象产品  及  抽象工厂方法  的数量
	从一个变多个，使具体工厂生成一组具体产品；

**优点**：使具体工厂可承担更多任务

**缺点**：若要加一种新的抽象产品，所有工厂都要修改，会违反开闭原则

### 3.3 建造者模式

**结构**：

- 产品角色(Product)：要创建的复杂对象；装有属性与属性对应的get，set方法

- 抽象建造者(Builder)：提供创建方法模板的接口（抽象类/接口）

- 具体建造者(ConcreteBuilder)：实现Builder接口，提供产品各个属性的创建方法，并提供 getResult() 方法返回装好的产品实例

  （增添额外的产品只需要添加具体建造者类）

- 指挥者(Director)：在指挥者里调用具体建造者来直接完成产品的组建，为客户端省去组建步骤并封装信息

**客户端代码**：

```java
Builder b = new ConcreteBuilder();
Director d = new Director(b);
Product p = d.construct();
```

**优点**：封装性好，增添相似的新产品容易，复杂产品的创建过程更清晰

**缺点**：产品必须要有很强的相似性，差异大的产品不能使用同一套建造者模式创建

### 3.4 原型模式

**结构**：

- 抽象原型类：为所有子类或实现类提供 clone() 方法的接口（父类或接口）
- 具体原型类：继承或实现了抽象原型类的 clone() 方法
- 客户类：客户端的使用者

**浅克隆**：只克隆普通成员变量，引用类型地址相同，成员对象相同

**深克隆**：也克隆引用类型，成员对象克隆一个相同的副本

### 3.5 单例模式

**结构**：单例类（每个类只能有一个对应的实例）
